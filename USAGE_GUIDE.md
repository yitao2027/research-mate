# ResearchMate 使用指南

## 🎯 功能特性

### 核心功能
1. **交互式采集** - 通过 3 个关键问题精准定位素材需求
2. **反幻觉机制** - 严格验证数据真实性，杜绝 AI 胡编乱造
3. **Word/PDF导出** - 生成专业素材报告（含提纲、总结、评估表）
4. **素材评估** - 四维度评分（可信度、时效性、完整性、交叉验证）
5. **反馈收集** - 支持用户指出需要补充的素材方向

## 🚀 快速开始

### 方式一：交互模式（推荐）
```bash
python main.py
```

系统会依次询问：

**问题 1：选题描述**
```
❓ 请描述您的选题（想写什么主题的文章）：
> 人工智能在医疗行业的应用与商业机会
```

**问题 2：关键实体**
```
💡 为了更精准地采集素材，请提供以下信息：
   关键词（产品名称、企业名称、人物姓名或事件名称，多个用逗号分隔）：
> 腾讯觅影，阿里健康，推想科技，AI 影像诊断
```

**问题 3：目标字数**
```
📝 您计划写多少字的内容？（例如：3000）：
> 5000
```

系统会自动计算需要采集的素材量（8-10 倍）：
- 目标字数：5000 字
- 预计采集：40000-50000 字符素

## 🔍 反幻觉机制

ResearchMate 通过以下机制确保素材真实性：

### 1. 数据验证（4 项检查）
- ✅ **数值检查** - 是否包含具体数据
- ✅ **主体检查** - 是否有明确公司/产品名
- ✅ **时间检查** - 是否有清晰时间标识
- ✅ **来源检查** - 是否标注数据来源

### 2. 置信度评分
```python
置信度 = (数值 + 主体 + 时间 + 来源) / 4

if 置信度 < 0.5:
    标记为"疑似幻觉数据"并剔除
```

### 3. 来源追溯
- 所有数据必须附带原始 URL
- 权威来源白名单加分（官方 > 券商 > 媒体）
- 无法验证来源的数据自动降级

## 📝 素材评估维度

### 四维度评分体系

| 维度 | 权重 | 评估标准 |
|------|------|----------|
| **来源可信度** | 40% | 官方披露 > 主流财经媒体 > 科技媒体 > 自媒体 |
| **时效性** | 25% | 3 天内 > 7 天内 > 15 天内 > 30 天内 |
| **数据完整性** | 20% | 有具体数值 > 有趋势描述 > 仅有定性描述 |
| **交叉验证** | 15% | 多源验证 > 单一来源但有背书 > 无验证 |

### 评级标准
- **S 级** (90-100 分)：官方财报 + 近期 + 数据完整 + 多源验证
- **A 级** (80-89 分)：券商研报 + 7 天内 + 有关键数据
- **B 级** (70-79 分)：主流媒体报道 + 15 天内 + 信息可靠
- **C 级** (<70 分)：来源不明或数据缺失

## 💬 反馈收集流程

采集完成后，系统会询问：

### Step 1：展示高价值素材
```
⭐ 评分最高的 10 条素材：
  1. [S] 腾讯 2025Q4 财报：AI 医疗营收同比增长 156% (评分：95.2)
  2. [A] 中信证券：AI 影像诊断市场规模突破 200 亿 (评分：88.5)
  ...
```

### Step 2：询问补充需求
```
❓ 是否还需要补充某些方面的素材？(y/n)：y

💡 请描述需要补充的素材类型或方向：
   （输入空行结束）：
> 希望补充更多海外企业的案例，比如谷歌 DeepMind 在医疗领域的布局
> 需要一些失败案例或风险提示
> 
```

### Step 3：满意度调查
```
❓ 当前素材是否满足写作需求？(满意素材数/15)：12
```

如果用户表示需要补充素材，系统会：
1. 记录补充方向
2. 询问是否立即重新采集
3. 针对薄弱方向定向搜索

## 🛠️ 高级用法

### 自定义采集源
编辑 `config.yaml`：
```yaml
sources:
  tech_media:
    enabled: true
    sites:
      - 36kr.com
      - huxiu.com
      - tmtpost.com
  
  industry_reports:
    enabled: true
    min_quality: A
  
  social_media:
    enabled: false  # 默认关闭，可信度较低
```

### 调整质量阈值
```yaml
filters:
  min_credibility_score: 0.8  # 最低可信度
  max_age_days: 14            # 最大年龄
  require_data_points: true   # 必须有具体数据
```

### 批量采集多个主题
创建 `topics.txt`：
```
人工智能在金融风控中的应用
新能源汽车产业链投资机会
光伏行业技术迭代与竞争格局
```

执行：
```bash
python main.py --topics-file topics.txt --output batch_2024/
```

## ❓ 常见问题

### Q1: 为什么有些素材被过滤了？
A: 系统会自动过滤低质量素材（<70 分），常见原因：
- 来源不明（自媒体、论坛）
- 时间过久（>30 天）
- 缺乏具体数据（只有观点没有事实）
- 无法交叉验证

### Q2: 如何增加采集数量？
A: 三种方式：
1. 扩大时间范围：`--days 30`
2. 降低质量阈值：修改 `config.yaml` 中 `min_credibility_score`
3. 添加更多采集源：启用 `social_media`

### Q3: Word 文档乱码怎么办？
A: 确保已安装中文字体：
```bash
# macOS 无需额外操作
# Windows 安装微软雅黑字体
```

### Q4: 如何引用素材？
A: Word 报告中每条素材都包含：
- 原文链接 URL
- 来源机构
- 发布日期
- 采集时间

写作时直接复制 URL 作为脚注即可。

## 🎓 最佳实践

### 1. 提问技巧
- **选题描述**：尽量具体，如"AI 在医疗影像诊断中的商业化进展"而非"AI 医疗"
- **关键词**：包含公司名、产品名、人名，如"腾讯觅影，陈广乾，A 轮融资"
- **字数规划**：根据文章类型设定
  - 短讯：500-1000 字
  - 分析文：3000-5000 字
  - 深度报告：8000-15000 字

### 2. 素材筛选
优先采用：
- ✅ S 级/A 级素材（评分≥80）
- ✅ 近 7 天内发布
- ✅ 官方财报/券商研报
- ✅ 包含具体数值和增长率

谨慎使用：
- ⚠️ C 级素材（评分<70）
- ⚠️ 单一来源且无背书
- ⚠️ 只有观点没有数据支撑

### 3. 补充采集
当用户反馈"需要更多 XX 方面素材"时：
1. 查看评估表中"补充建议"列
2. 针对性调整关键词
3. 扩大时间范围或启用新采集源
